# Git-Mod Documentation

Welcome to the **Git-Mod** documentation! Git-Mod is a powerful CLI supercharger for Git, designed to streamline advanced workflows, manage multiple identities, and provide interactive tools for repository maintenance.

Browse the documentation by category:

## 👤 [Identity & Configuration](identity.md)
Everything related to who you are in your repositories.
- `git mod profile`: Identity profiles and SSH keys.
- **🤖 Magic Identity**: How to auto-switch identities based on directory paths.
- `git mod remote`: Remote repository management.
- `git mod init`: Smart initialization.

## 🛠️ [Daily Workflow](daily-workflow.md)
Commands you'll use every day to keep your work organized.
- `git mod commit`: The Conventional Commits wizard.
- `git mod status`: Fancy dashboard for your repository.
- `git mod stash`: Interactive stash manager.
- `git mod merge`: Smart merge with squash options.
- `git mod rollback`: The definitive time machine.
- `git mod ignore` / `oops`: The oops fixer for accidentally tracked files.
- `git mod changelog`: Generate CHANGELOG.md from Conventional Commits.

## 🚀 [Advanced Operations](advanced.md)
Heavy-duty commands for complex repositories and fixing major issues.
- `git mod rescue`: Browse reflog to restore lost commits.
- `git mod hunt`: Visual `git bisect` to find bugs.
- `git mod checkpoint`: Create named snapshots before risky changes.
- `git mod subtree`: Interactive external repository manager.
- `git mod worktree`: Context switching made easy.
- `git mod sparse`: Optimize large repositories with sparse-checkout.
- `git mod optimize`: Automated cleanup and maintenance tasks.
- `git mod sync`: One-go synchronization with remotes.
- `git mod clean`: Interactive branch cleanup.
